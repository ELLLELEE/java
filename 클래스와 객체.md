# 05 클래스와 객체
### - 객체지향 프로그램
- 객체 (데이터 + 메서드) 간의 상호작용을 통해 전체 시스템 구성
- 누가 이 일을 수행할 것인가 (who) -> 명사(객체) 에 집중
- 절차 vs 객체
  - 절차
    - 데이터와 함수 분리되어있음
    - 관리하기 빡셈
  - 객체
    - 데이터와 기능(함수)이 객체 단위로 묶여 상호작용
#### - 객체 지향의 5대 특징
      - 캡슐화 -> 데이터와 메서드 하나의 객체로 묶는 것
      - 추상화 -> 객체의 핵심적인 특징을 골라내 단순화
      - 다형성 -> 같은 명령어에 대해 객체마다 각자의 방식으로 동작
      - 정보은닉 -> 객체 내부의 민감한 데이터나 로직 외부로 숨김
      - 상속 -> 부모 객체의 기능 물려받음

### - 클래스
  - 필드
    - 객체의 상태 저장하는 변수 -> String color;
  -메서드
      - 객체의 행동 정의하는 함수 -> void break()
- 클래스 정의 방법
  - class 키워드 사용
- 클래스 명명 규칙
  - 첫글자 무조건 대문자 단어가 이어질때마다 첫글자 대문자
  - 명사 사용
  - 파일 이름 = public 클래스 이름

### - 객체(instance)
- 객체 생성할 때 new 키워드 사용 -> SmartSensor sensor1 = new SmartSensor();
  1) SmartSensor sensor1 : sensor1이라는 이름의 리모컨을 스택에 저장
  2) new SmartSensor() : 메모리의 빈공간(heap)을 찾아 실제 센서 객체를 생성
  3) = : 생성된 객체의 주소를 sensor1변수에 저장하여 연결

- 객체의 필드나 메서드에 접근할 때 '.'을 사용
  - sensor1.sensorId = "lee";
