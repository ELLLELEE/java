# 02 변수 상수 연산자
## - 변수란?
  - 값을 저장할 수 있는 메모리 상의 공간
1) 변수 선언
   - 이런 데이터 준비해줘라고 컴퓨터에 알림
   - int prodctionCount ;
2) 변수 할당
   - 준비된 변수에 값을 넣는 과정
   - productionCount = 100;
3) 선언 & 할당
   - int productionCount = 100;

1) 변수 이름짓기
  - 필수
    - 대소문자 구분
    - 영문자, 숫자, 언더바, 달러기호만 사용가능
    - 숫자 시작금지
    - 예약어(int, class) 이미 의미가 정해진 단어는 사용못함
  - 권장
    - 의미있는 이름 사용
    - 여러단어 조합 두번째 단어의 첫글자 대문자 ex) maxSpeed
    - 참, 거짓 나타내는 변수는 is,has 붙이기
2) 클래스 이름짓기
  - ex) MemberService -> 대문자 시작, 명사중심
3) 메서드 이름짓기
  - ex) saveMenber -> 소문자 시작, 동사중심
4) 변수
  - 소문자 시작, 명사중심
5) 상수
  - ex) MAX_COUNT -> 전체 대문자, 언더바 연결

### * 꼭 변수타입 지정

## - 상수란
  - 변하지 않는 값 저장하는 공간
  - final 붙여서 사용
  - final int MAX_SPPED = 10;
- 상수를 사용하는 이유
  - 값이 변경 안되기때문에 안전
  - 의미가 명확함
  - 유지보수 용이성
- 지역 상수 & 전역 상수
  - 지역상수
    - 메서드 내부에서 선언 해당 메서드 실행중에만 잠시 사용되는 상수
    - final int COUNT =10;
  - 전역 상수
    - 클래스 레벨에서 선언 클래스 내부 혹은 외부에서 접근 가능
    - public fine int MAX_NUM = 100;

## - 자바의 자료형
- 기본형
  - 8개의 기본 타입 변수가 가르키는 메모리에 직접 값 저장
- 참조형
  - 변수에는 실제 데이터가 있는 위치를 가르키는 레퍼런스 정보
  - 두 영역에서 관리
  - stack -> 변수의 이름과 실제 데이터가 저장된 주소
  - heap -> 실제값인 객체 데이터 저장
  - 배열, class, interface 등등 있음
  - int[]

- 자료형 선택방법
  - 정수는 int , 21억 넘으면 long
  - 실정수는 double,
  - 금융 데이터는 오차에 민감함으로 BigCecimal 사용
  - 0.1은 2진수에서 무한소수가 됨
    
## - 형변환
- 변수의 타입을 변환
- 크기가 작은 타입에서 큰 타입으로 자동 변환 가능
- byte(1) -> int(4) -> long(8) -> float(4) -> double(8)
- float은 4바이트지만 표현번위가 long보다 넓어 더 큰 그릇으로 취급
- but 형변환 일어날때 정밀도 떨어질수도

- 큰크기 -> 작은 크기 : 데이터 손실
#### * 나눌땐 하나를 double로 강제 변환해야 정확한 실수 나옴 -> double 95.0 / 2 -> 47.5

- 비교연산자

## - 서식 지정자
- 데이터를 어떤 형식으로 출력할지 지정하는 예약어, %로 시작
  - %d -> 정수 -> int, long
  - %f -> 실수 -> float, double
  - %s -> 문자열 -> 텍스트
  - %c -> 문자 -> char타입
  - %n -> 줄바꿈
  - ex) -> System.out.printf("CH%2d 전압 : %.2fv %n, channel, voltage)
